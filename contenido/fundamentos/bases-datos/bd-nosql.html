Bases de Datos NoSQL

Las bases de datos NoSQL (Not Only SQL) son sistemas de gestión de bases de datos no relacionales diseñados para manejar grandes volúmenes de datos no estructurados o semi-estructurados. Son ideales para aplicaciones modernas que requieren alta escalabilidad, flexibilidad y rendimiento. A continuación, se presenta una guía detallada sobre las bases de datos NoSQL más populares, sus características principales, y cómo utilizarlas en proyectos modernos.

Bases de datos NoSQL populares

MongoDB

MongoDB es una base de datos orientada a documentos, diseñada para almacenar datos en formato JSON-like (BSON). Es ampliamente utilizada en aplicaciones web y móviles.

Características principales:

  • Orientado a documentos: Almacena datos en documentos flexibles (BSON).
  • Escalabilidad horizontal: Soporte para sharding y replicación.
  • Consultas flexibles: Soporte para consultas complejas y agregaciones.
  • Índices: Índices secundarios para mejorar el rendimiento de las consultas.

Ejemplo básico:

// Insertar un documento
db.empleados.insertOne({
    nombre: "Juan Pérez",
    salario: 50000,
    departamento: "TI"
});

// Consultar documentos
db.empleados.find({ departamento: "TI" });

Cassandra

Cassandra es una base de datos distribuida diseñada para manejar grandes volúmenes de datos en múltiples servidores sin un punto único de fallo.

Características principales:

  • Alta disponibilidad: Sin punto único de fallo, gracias a su arquitectura distribuida.
  • Escalabilidad lineal: Añade más nodos para aumentar la capacidad.
  • Modelo de columnas: Almacena datos en columnas en lugar de filas.
  • Consistencia ajustable: Permite ajustar el nivel de consistencia según las necesidades.

Ejemplo básico:

-- Crear un keyspace
CREATE KEYSPACE empresa WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1};

-- Crear una tabla
CREATE TABLE empresa.empleados (
    id UUID PRIMARY KEY,
    nombre TEXT,
    salario DECIMAL
);

-- Insertar datos
INSERT INTO empresa.empleados (id, nombre, salario) VALUES (uuid(), 'Ana Gómez', 45000);

-- Consultar datos
SELECT * FROM empresa.empleados;

Redis

Redis es una base de datos en memoria que se utiliza como almacén de clave-valor, caché y broker de mensajes. Es conocido por su alto rendimiento.

Características principales:

  • Almacenamiento en memoria: Extremadamente rápido para operaciones de lectura/escritura.
  • Estructuras de datos avanzadas: Soporta strings, hashes, listas, conjuntos, y más.
  • Persistencia opcional: Puede persistir datos en disco si es necesario.
  • Pub/Sub: Soporta mensajería publish/subscribe.

Ejemplo básico:

# Insertar un valor
SET nombre "Juan Pérez"

# Obtener un valor
GET nombre

# Usar un hash
HSET empleado:1 nombre "Ana Gómez" salario 45000
HGETALL empleado:1

Couchbase

Couchbase es una base de datos NoSQL que combina las características de una base de datos de documentos con las de una base de datos clave-valor.

Características principales:

  • Modelo de documentos: Almacena datos en formato JSON.
  • Escalabilidad horizontal: Soporte para clustering y replicación.
  • Consultas N1QL: Lenguaje de consultas similar a SQL para documentos JSON.
  • Integración con móviles: Soporte para sincronización de datos en aplicaciones móviles.

Ejemplo básico:

-- Insertar un documento
INSERT INTO empresa.empleados (KEY, VALUE)
VALUES ("1", { "nombre": "Carlos Ruiz", "salario": 55000 });

-- Consultar documentos
SELECT * FROM empresa.empleados WHERE salario > 50000;

Elasticsearch

Elasticsearch es un motor de búsqueda y análisis distribuido, diseñado para búsquedas en tiempo real y análisis de grandes volúmenes de datos.

Características principales:

  • Búsqueda en tiempo real: Ideal para aplicaciones que requieren búsquedas rápidas.
  • Escalabilidad horizontal: Distribuye datos y consultas en múltiples nodos.
  • Análisis de datos: Soporte para agregaciones y análisis complejos.
  • Integración con Kibana: Herramienta de visualización de datos.

Ejemplo básico:

// Insertar un documento
POST /empleados/_doc/1
{
    "nombre": "Laura Díaz",
    "salario": 60000,
    "departamento": "TI"
}

// Consultar documentos
GET /empleados/_search
{
    "query": {
        "match": {
            "departamento": "TI"
        }
    }
}

Neo4j

Neo4j es una base de datos de grafos diseñada para manejar datos altamente conectados, como redes sociales, recomendaciones y análisis de relaciones.

Características principales:

  • Modelo de grafos: Almacena datos como nodos y relaciones.
  • Consultas en Cypher: Lenguaje de consultas específico para grafos.
  • Análisis de relaciones: Ideal para aplicaciones que requieren análisis de conexiones.
  • Escalabilidad: Soporte para clustering y replicación.

Ejemplo básico:

// Crear un nodo
CREATE (juan:Empleado { nombre: "Juan Pérez", salario: 50000 });

// Crear una relación
CREATE (ana:Empleado { nombre: "Ana Gómez", salario: 45000 });
CREATE (juan)-[:TRABAJA_CON]->(ana);

// Consultar datos
MATCH (e:Empleado) RETURN e;
