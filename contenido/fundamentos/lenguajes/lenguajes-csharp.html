C#

C# (pronunciado "C Sharp") es un lenguaje de programación moderno, orientado a objetos y fuertemente tipado, desarrollado por Microsoft en el año 2000. Es ampliamente utilizado en el desarrollo de aplicaciones de escritorio, web, móviles, juegos (con Unity) y servicios en la nube. C# es parte del ecosistema .NET y es conocido por su sintaxis limpia, robustez y versatilidad.

Características principales

Fundamentos

  • Orientado a objetos: Soporta clases, herencia, polimorfismo y encapsulamiento.
  • Fuertemente tipado: Los tipos de datos deben ser declarados explícitamente.
  • Moderno y seguro: Incluye características como recolector de basura y manejo de excepciones.
  • Multiplataforma: Gracias a .NET Core y .NET 5+ (ahora .NET 6+).
  • Interoperabilidad: Puede interactuar con código C++, F# y otros lenguajes.
  • Asincronía nativa: Soporte para programación asíncrona con async y await.

Paradigmas soportados

  • Orientado a objetos: Clases, interfaces, herencia, etc.
  • Imperativo: Secuencias de declaraciones que cambian el estado.
  • Funcional: Soporta funciones de primera clase, expresiones lambda y LINQ.
  • Concurrente: Programación asíncrona y paralelismo con TPL (Task Parallel Library).

Evolución de C#

Historia y versiones principales

  • 2000: Lanzamiento de C# 1.0 con .NET Framework 1.0.
  • 2005: C# 2.0 introdujo genéricos, iteradores y tipos anulables.
  • 2007: C# 3.0 introdujo LINQ, expresiones lambda y tipos anónimos.
  • 2012: C# 5.0 introdujo async y await para programación asíncrona.
  • 2019: C# 8.0 introdujo rangos, índices, tipos de referencia anulables y patrones de coincidencia.
  • 2021: C# 10 introdujo mejoras en registros, tipos anónimos y expresiones lambda.
  • 2023: C# 11 introdujo patrones de coincidencia mejorados y tipos genéricos más flexibles.

Características modernas (C# 8.0+)

Sintaxis y estructuras

// Expresiones lambda
Func<int, int> square = x => x * x;

// LINQ (Language Integrated Query)
var numbers = new List<int> { 1, 2, 3, 4, 5 };
var evenNumbers = numbers.Where(n => n % 2 == 0);

// Tipos de referencia anulables

// Rangos e índices

Asincronía moderna

// Async/Await
public async Task<string> FetchDataAsync()
{
    using var client = new HttpClient();
    var response = await client.GetStringAsync("https://api.example.com/data");
    return response;
}

// Paralelismo con TPL
Parallel.For(0, 10, i => Console.WriteLine(i));

Entornos de ejecución

.NET Framework

El entorno de ejecución original para aplicaciones Windows. Aunque sigue siendo utilizado, ha sido reemplazado en gran medida por .NET Core y .NET 5+.

Características:

  • Compatibilidad con aplicaciones de escritorio (Windows Forms, WPF).
  • Servicios web (ASP.NET Web Forms, WCF).
  • Limitado a Windows.

.NET Core / .NET 5+

La evolución moderna de .NET, multiplataforma y de alto rendimiento. .NET 5 unificó .NET Core y .NET Framework.

Características:

  • Multiplataforma: Windows, Linux, macOS.
  • Alto rendimiento y escalabilidad.
  • Soporte para aplicaciones web (ASP.NET Core), servicios en la nube y microservicios.
  • Compatibilidad con contenedores (Docker).

Programación asíncrona

Evolución del manejo asíncrono

Threads (tradicional)

// Creación de hilos
Thread thread = new Thread(() => Console.WriteLine("Hilo en ejecución"));
thread.Start();

Async/Await (moderno)

public async Task<string> FetchDataAsync()
{
    using var client = new HttpClient();
    var response = await client.GetStringAsync("https://api.example.com/data");
    return response;
}

Paralelismo con TPL

// Parallel.For
Parallel.For(0, 10, i => Console.WriteLine(i));

// Task Parallel Library (TPL)
Task.Run(() => Console.WriteLine("Tarea en segundo plano"));

Ecosistema y herramientas

Frameworks principales

ASP.NET Core

  • Framework para aplicaciones web y APIs.
  • Multiplataforma y de alto rendimiento.
  • Soporte para MVC, Razor Pages y Web API.

Entity Framework Core

  • ORM (Object-Relational Mapping) para bases de datos.
  • Soporte para SQL Server, PostgreSQL, MySQL, SQLite, etc.
  • Migrations para gestión de esquemas de bases de datos.

Herramientas de desarrollo

Visual Studio

  • IDE completo para desarrollo en C# y .NET.
  • Depuración avanzada, refactorización y pruebas integradas.
  • Soporte para aplicaciones web, móviles, y de escritorio.

Visual Studio Code

  • Editor ligero y multiplataforma.
  • Extensible con plugins para C# y .NET.
  • Ideal para desarrollo rápido y ligero.

Mejores prácticas

Arquitectura y organización de código

Patrones de diseño comunes

  • Singleton: Garantiza una única instancia de una clase.
  • Factory: Encapsula la creación de objetos.
  • Repository: Aísla la lógica de acceso a datos.
  • Dependency Injection: Inyección de dependencias para un código más modular.

Estructura de proyectos

src/
├── Application/        # Lógica de negocio
│   ├── Services/
│   └── Models/
├── Infrastructure/     # Acceso a datos y servicios externos
│   ├── Repositories/
│   └── Data/
├── Web/                # Capa de presentación (ASP.NET Core)
│   ├── Controllers/
│   ├── Views/
│   └── wwwroot/
└── Tests/              # Pruebas unitarias y de integración
    ├── UnitTests/
    └── IntegrationTests/

Performance

  • Uso de async/await: Evitar bloqueos en operaciones I/O.
  • Optimización de consultas: Usar LINQ eficientemente y evitar N+1 queries.
  • Caching: Implementar caché para datos frecuentemente accedidos.
  • Minimizar allocations: Reducir la creación de objetos innecesarios.

Seguridad

  • Validación de entradas: Evitar inyecciones SQL y XSS.
  • Autenticación y autorización: Usar Identity para gestión de usuarios.
  • HTTPS: Proteger las comunicaciones con cifrado.
  • Protección de datos: Usar cifrado para datos sensibles.
