Java

Java es un lenguaje de programación de propósito general, orientado a objetos y fuertemente tipado, desarrollado por Sun Microsystems (ahora propiedad de Oracle) en 1995. Es conocido por su portabilidad, robustez y amplia adopción en aplicaciones empresariales, desarrollo web, aplicaciones móviles y más.

Características principales

Fundamentos

  • Orientado a objetos: Todo en Java es un objeto, con excepción de los tipos primitivos.
  • Fuertemente tipado: Los tipos de datos deben ser declarados explícitamente.
  • Portabilidad: "Write once, run anywhere" gracias a la JVM (Java Virtual Machine).
  • Recolección de basura: Gestión automática de memoria.
  • Multihilo: Soporte nativo para programación concurrente.
  • Seguridad: Diseñado con características de seguridad integradas.

Paradigmas soportados

  • Orientado a objetos: Clases, herencia, polimorfismo, encapsulamiento.
  • Imperativo: Secuencias de declaraciones que cambian el estado.
  • Funcional: Introducido en Java 8 con lambdas y streams.
  • Concurrente: Programación multihilo y manejo de hilos.

Evolución de Java

Historia y versiones principales

  • 1995: Lanzamiento de Java 1.0 por Sun Microsystems.
  • 2004: Java 5 (Java SE 5.0) introdujo genéricos, enumeraciones, y autoboxing.
  • 2014: Java 8 introdujo lambdas, streams, y la API de fecha y hora.
  • 2017: Java 9 introdujo módulos (Project Jigsaw).
  • 2020: Java 15 introdujo text blocks y mejoras en la recolección de basura.
  • 2023: Java 21 introdujo mejoras en concurrencia y patrones de diseño.

Entornos de ejecución

JVM (Java Virtual Machine)

La JVM es el entorno de ejecución que permite a Java ser portable. Ejecuta bytecode compilado desde código fuente Java.

Características:

  • Portabilidad: "Write once, run anywhere".
  • Recolección de basura: Gestión automática de memoria.
  • Optimización en tiempo de ejecución: Just-In-Time (JIT) compilation.
  • Seguridad: Sandboxing y verificación de bytecode.

Ejemplo de compilación y ejecución:

# Compilar
javac HelloWorld.java

# Ejecutar
java HelloWorld

JDK (Java Development Kit)

El JDK incluye herramientas para desarrollar, compilar y depurar aplicaciones Java.

Características:

  • Compilador (javac).
  • Herramientas de depuración (jdb).
  • Javadoc: Generación de documentación.
  • JAR: Empaquetado de aplicaciones.

Ejemplo de uso de JAR:

# Crear un JAR
jar cf myapp.jar -C classes .

# Ejecutar un JAR
java -jar myapp.jar

Programación asíncrona

Evolución del manejo asíncrono

Threads (tradicional)

// Creación de hilos
Thread thread = new Thread(() -> {
    System.out.println("Hilo en ejecución");

thread.start();

CompletableFuture (Java 8+)

CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    return "Resultado asíncrono";

// Parallel Streams
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
names.parallelStream().forEach(System.out::println);

Ecosistema y herramientas

Gestores de dependencias

Maven

  • Gestión de dependencias y construcción de proyectos.
  • Configuración mediante archivos XML (pom.xml).
  • Gran repositorio central (Maven Central).

Gradle

  • Gestión de dependencias y construcción basada en Groovy/Kotlin DSL.
  • Mayor flexibilidad y rendimiento que Maven.
  • Compatibilidad con Maven Central y otros repositorios.

Frameworks principales

Spring

  • Framework completo para aplicaciones empresariales.
  • Inyección de dependencias, MVC, seguridad, etc.
  • Spring Boot para aplicaciones rápidas y autoconfiguradas.

Hibernate

  • ORM (Object-Relational Mapping) para bases de datos.
  • Simplifica la interacción con bases de datos relacionales.
  • Integración con JPA (Java Persistence API).

Mejores prácticas

Arquitectura y organización de código

Patrones de diseño comunes

  • Singleton: Garantiza una única instancia de una clase.
  • Factory: Encapsula la creación de objetos.
  • Observer: Notifica cambios a múltiples observadores.
  • Strategy: Permite cambiar algoritmos en tiempo de ejecución.

Estructura de proyectos

src/
├── main/
│   ├── java/            # Código fuente
│   │   ├── com/         # Paquetes
│   │   │   └── example/
│   │   │       ├── model/
│   │   │       ├── service/
│   │   │       └── controller/
│   │   └── resources/   # Recursos (configuraciones, propiedades)
│   └── webapp/          # Recursos web (HTML, CSS, JS)
└── test/                # Pruebas unitarias
    └── java/            # Código de pruebas

Performance

  • Uso de colecciones adecuadas: Elegir la estructura de datos correcta (List, Set, Map).
  • Evitar bloqueos en hilos: Usar técnicas de concurrencia como CompletableFuture.
  • Optimización de memoria: Minimizar el uso de objetos innecesarios.
  • Uso de streams: Aprovechar las operaciones funcionales para procesamiento de datos.
