JavaScript

JavaScript es un lenguaje de programación interpretado, de alto nivel, dinámicamente tipado y multiparadigma. Originalmente creado para agregar interactividad a las páginas web, ha evolucionado hasta convertirse en uno de los lenguajes más versátiles y utilizados, con aplicaciones en frontend, backend, aplicaciones móviles, IoT y más.

Características principales

Fundamentos

  • Tipado dinámico: Las variables pueden cambiar de tipo durante la ejecución.
  • Orientado a objetos: Basado en prototipos, no en clases (aunque ES6+ añadió sintaxis de clases).
  • Funciones de primera clase: Las funciones pueden ser tratadas como cualquier otra variable.
  • Event-driven: Arquitectura basada en eventos y callbacks.
  • Single-threaded: Modelo de concurrencia basado en un solo hilo con un event loop.
  • Interpretado: No requiere compilación previa.

Paradigmas soportados

  • Imperativo: Secuencias de declaraciones que cambian el estado.
  • Orientado a objetos: A través de prototipos y, más recientemente, clases.
  • Funcional: Soporte para funciones de orden superior, mapeo, filtrado, etc.
  • Orientado a eventos: Callbacks, promesas, async/await.
  • Meta-programación: A través de proxies y reflection.

Evolución de JavaScript

Historia y versiones principales

  • 1995: Creado por Brendan Eich en Netscape bajo el nombre de Mocha, luego LiveScript.
  • 1997: Estandarizado como ECMAScript (ES1).
  • ES3 (1999): Introdujo expresiones regulares, try/catch.
  • ES5 (2009): Añadió strict mode, métodos de array (map, filter, etc.), JSON.
  • ES6/ES2015: Mayor actualización, con clases, módulos, promesas, arrow functions.
  • ES2016-2022: Actualizaciones anuales con nuevas características.

Entornos de ejecución

Navegadores

Entorno original de JavaScript, con acceso al DOM (Document Object Model) y Web APIs.

Características:

  • Acceso al DOM para manipulación de HTML/CSS.
  • APIs web: fetch, localStorage, WebSockets, WebGL, etc.
  • Seguridad basada en Same-Origin Policy.
  • Limitaciones de rendimiento (single-threaded).
  • Web Workers para operaciones en segundo plano.

Ejemplo de manipulación DOM:

// Selección de elementos
const button = document.querySelector('#myButton');
const items = document.querySelectorAll('.item');

// Manipulación
button.addEventListener('click', () => {
const div = document.createElement('div');
div.textContent = 'Nuevo elemento';
div.classList.add('new-item');
document.body.appendChild(div);
});

// Estilos y clases
element.style.color = 'red';
element.classList.add('active');
element.classList.toggle('visible');

Node.js

Entorno de ejecución de JavaScript del lado del servidor basado en el motor V8 de Chrome.

Características:

  • Acceso al sistema de archivos.
  • Módulos CommonJS/ES (require/import).
  • APIs para networking, streams, eventos.
  • Event-loop no bloqueante.
  • NPM (Node Package Manager).
  • Bueno para APIs, microservicios, aplicaciones en tiempo real.

Ejemplo de servidor HTTP:

// Servidor básico con Node.js
const http = require('http');

const server = http.createServer((req, res) => {
res.statusCode = 200;
res.setHeader('Content-Type', 'text/plain');
res.end('Hello World\n');
});

server.listen(3000, '127.0.0.1', () => {
console.log('Servidor ejecutándose en http://127.0.0.1:3000/');
});

// Manejo de archivos
const fs = require('fs');
fs.readFile('archivo.txt', 'utf8', (err, data) => {
if (err) throw err;
console.log(data);
});

Programación asíncrona

Evolución del manejo asíncrono

Callbacks (tradicional)

function fetchData(callback) {
setTimeout(() => {
  const data = { name: 'Juan', age: 30 };
  callback(null, data);
}, 1000);
}

fetchData((error, data) => {
if (error) {
  console.error(error);
  return;
}
console.log(data);

// Callback hell
fetchRelatedData(data.id, (error, relatedData) => {
  // Más callbacks anidados...
});
});

Promesas (ES6)

function fetchData() {
return new Promise((resolve, reject) => {
  setTimeout(() => {
    const data = { name: 'Juan', age: 30 };
    resolve(data);
    // En caso de error: reject(error)
  }, 1000);
});
}

fetchData()
.then(data => {
  console.log(data);
  return fetchRelatedData(data.id);
})
.then(relatedData => {
  console.log(relatedData);
})
.catch(error => {
  console.error(error);
});

Patrones de concurrencia

Control de flujo

// Ejecución secuencial
async function sequential() {
const results = [];
for (const id of ids) {
  const result = await fetchItem(id);
  results.push(result);
}
return results;
}

// Ejecución en paralelo (limitada)
async function parallelWithLimit(urls, limit = 3) {
const results = [];
const batches = [];

for (let i = 0; i < urls.length; i += limit) {
  const batch = urls.slice(i, i + limit).map(
    url => fetch(url).then(res => res.json())
  );
  batches.push(batch);
}

for (const batch of batches) {
  results.push(...await Promise.all(batch));
}

return results;
}

Gestión de errores

// Retry Pattern
async function fetchWithRetry(url, retries = 3) {
for (let i = 0; i < retries; i++) {
  try {
    return await fetch(url);
  } catch (err) {
    if (i === retries - 1) throw err;
    console.log(`Retry ${i+1}/${retries}`);
    await new Promise(r => setTimeout(r, 1000));
  }
}
}

// Promise timeout
function timeoutPromise(promise, ms) {
const timeout = new Promise((_, reject) => {
  setTimeout(() => reject(
    new Error('Timeout')), ms);
});
return Promise.race([promise, timeout]);
}

Ecosistema y herramientas

Gestores de paquetes

NPM (Node Package Manager)

  • Gestor de paquetes estándar para Node.js.
  • Enorme repositorio de paquetes (1.3M+).
  • package.json para gestión de dependencias.
  • Scripts personalizados.

Yarn

  • Mayor velocidad que NPM.
  • Caching mejorado y determinismo.
  • Verificación de integridad.
  • Workspaces para monorepos.

Bundlers y build tools

Webpack

  • Empaquetador completo y configurable.
  • Sistema de loaders y plugins.
  • Code splitting.
  • Hot Module Replacement.

Vite

  • Herramienta de desarrollo ultrarrápida.
  • Desarrollo sin bundling (ESM nativo).
  • Compilación con Rollup.
  • HMR optimizado.

Frameworks y librerías principales

Frontend

React

  • Biblioteca para construir interfaces.
  • Basado en componentes.
  • Virtual DOM.
  • Flujo unidireccional de datos.
  • Hooks para lógica reutilizable.
  • Ecosistema extenso (React Router, Redux, etc.).

Vue.js

  • Framework progresivo para UI.
  • Sistema de reactividad simple.
  • Sintaxis de plantillas HTML.
  • Options API y Composition API.
  • Herramientas oficiales (Vue Router, Pinia).
  • Escalable de simple a complejo.

Backend

Express.js

  • Framework web minimalista.
  • Sistema de middleware.
  • Enrutamiento.
  • Plantillas de vistas.
  • Gran ecosistema de plugins.
  • Base para muchos otros frameworks.

NestJS

  • Framework inspirado en Angular.
  • Arquitectura modular.
  • TypeScript por defecto.
  • Inyección de dependencias.
  • Decoradores para metadatos.
  • Soporte para microservicios.

Mejores prácticas

Arquitectura y organización de código

Patrones de diseño comunes

  • Módulos: Organizar código en unidades reutilizables.
  • Fachada: Interfaz simplificada para subsistemas complejos.
  • Observador: Notificar cambios a múltiples observadores.
  • Singleton: Instancia única (con precaución).
  • Factory: Crear objetos sin especificar la clase exacta.
  • Provider: Inyección de dependencias.

Estructura de proyectos

src/
├── components/        # Componentes UI (React/Vue)
│   ├── common/        # Componentes reutilizables
│   └── features/      # Componentes específicos 
├── hooks/             # Lógica reutilizable
├── services/          # Comunicación con APIs
├── utils/             # Funciones útiles
├── store/             # Estado global
│   ├── slices/        # Redux Toolkit o similar
│   └── index.js       # Configuración de store
├── pages/             # Rutas/páginas
├── routes/            # Configuración de rutas
├── styles/            # Estilos globales
└── index.js           # Punto de entrada

Performance

  • Memoización: Caché de resultados de funciones puras (useMemo, React.memo).
  • Code splitting: Dividir código en chunks más pequeños para cargar bajo demanda.
  • Tree shaking: Eliminar código no utilizado del bundle.
  • Lazy loading: Cargar componentes solo cuando son necesarios.
  • Web workers: Operaciones pesadas en hilos separados.
  • Virtualización: Renderizar solo los elementos visibles en listas largas.
