Google Cloud Platform (GCP)

Google Cloud Platform (GCP) es una suite de servicios en la nube ofrecida por Google que proporciona infraestructura, almacenamiento, análisis de datos, machine learning, y más. GCP es conocido por su escalabilidad, rendimiento y herramientas avanzadas de análisis de datos. A continuación, se presenta una guía detallada sobre los servicios más populares de GCP, con ejemplos prácticos y casos de uso.

Servicios populares de GCP

Compute Engine

Compute Engine es un servicio de infraestructura como servicio (IaaS) que permite crear y gestionar máquinas virtuales en la nube.

Características principales:

  • Flexibilidad: Soporta múltiples sistemas operativos y configuraciones.
  • Escalabilidad: Aumenta o reduce la capacidad según la demanda.
  • Integración: Se integra con otros servicios de GCP como Storage, Networking, y más.

Ejemplo básico:

# Crear una máquina virtual en Compute Engine
gcloud compute instances create my-vm --machine-type n1-standard-1 --image-family debian-10 --image-project debian-cloud

App Engine

App Engine es una plataforma completamente gestionada para construir y escalar aplicaciones web y móviles.

Características principales:

  • Multiplataforma: Soporta aplicaciones en Python, Java, Node.js, Go, y más.
  • Escalabilidad automática: Escala automáticamente según la demanda.
  • Integración con CI/CD: Soporta integración continua y entrega continua.

Ejemplo básico:

# Desplegar una aplicación en App Engine
gcloud app deploy app.yaml

Cloud SQL

Cloud SQL es un servicio de base de datos relacional completamente gestionado que soporta MySQL, PostgreSQL y SQL Server.

Características principales:

  • Escalabilidad: Aumenta o reduce la capacidad según la demanda.
  • Alta disponibilidad: Ofrece replicación y copias de seguridad automáticas.
  • Seguridad avanzada: Incluye encriptación, auditoría y control de acceso.

Ejemplo básico:

# Crear una instancia de Cloud SQL
gcloud sql instances create my-instance --database-version=MYSQL_5_7 --tier=db-n1-standard-1 --region=us-central1

Cloud Functions

Cloud Functions es un servicio de computación sin servidor que permite ejecutar código en respuesta a eventos sin gestionar la infraestructura.

Características principales:

  • Sin servidor: No es necesario gestionar servidores.
  • Escalabilidad automática: Escala automáticamente según la demanda.
  • Integración con otros servicios de GCP: Se integra con Cloud Storage, Pub/Sub, y más.

Ejemplo básico:

// Ejemplo de Cloud Function en Node.js
exports.helloWorld = (req, res) => {
  res.send('Hello, World!');
};

Cloud Storage

Cloud Storage es un servicio de almacenamiento de objetos que permite almacenar y acceder a grandes cantidades de datos.

Características principales:

  • Almacenamiento de objetos: Almacena cualquier cantidad de datos.
  • Durabilidad y disponibilidad: Diseñado para una durabilidad del 99.999999999%.
  • Control de acceso: Permite gestionar permisos y políticas de acceso.

Ejemplo básico:

# Crear un bucket en Cloud Storage
gsutil mb gs://my-bucket

# Subir un archivo
gsutil cp myfile.txt gs://my-bucket/

BigQuery

BigQuery es un servicio de almacenamiento y análisis de datos completamente gestionado que permite ejecutar consultas SQL sobre grandes conjuntos de datos.

Características principales:

  • Análisis en tiempo real: Ejecuta consultas SQL sobre grandes volúmenes de datos en segundos.
  • Escalabilidad: Maneja petabytes de datos sin necesidad de gestionar la infraestructura.
  • Integración con otros servicios de GCP: Se integra con Cloud Storage, Dataflow, y más.

Ejemplo básico:

-- Ejecutar una consulta en BigQuery
SELECT name, COUNT(*) as count
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE state = 'TX'
GROUP BY name
ORDER BY count DESC
LIMIT 10;

Kubernetes Engine (GKE)

Google Kubernetes Engine (GKE) es un servicio gestionado que permite implementar y gestionar clústeres de Kubernetes en GCP.

Características principales:

  • Gestión simplificada: GCP gestiona los nodos maestros de Kubernetes.
  • Escalabilidad: Escala automáticamente los nodos de trabajo.
  • Integración con otros servicios de GCP: Se integra con Cloud Monitoring, Cloud Logging, y más.

Ejemplo básico:

# Crear un clúster de GKE
gcloud container clusters create my-cluster --num-nodes=3 --zone=us-central1-a

Cloud Pub/Sub

Cloud Pub/Sub es un servicio de mensajería que permite enviar y recibir mensajes entre aplicaciones de manera asíncrona.

Características principales:

  • Escalabilidad: Maneja millones de mensajes por segundo.
  • Fiabilidad: Garantiza la entrega de mensajes al menos una vez.
  • Integración con otros servicios de GCP: Se integra con Cloud Functions, Dataflow, y más.

Ejemplo básico:

# Crear un tema en Pub/Sub
gcloud pubsub topics create my-topic

# Publicar un mensaje
gcloud pubsub topics publish my-topic --message "Hello, Pub/Sub!"

Cloud AI Platform

Cloud AI Platform es un servicio que permite construir, entrenar y desplegar modelos de machine learning en la nube.

Características principales:

  • Entrenamiento distribuido: Soporta entrenamiento de modelos en grandes conjuntos de datos.
  • Despliegue de modelos: Permite desplegar modelos como servicios web.
  • Integración con TensorFlow: Soporta TensorFlow y otros frameworks de machine learning.

Ejemplo básico:

# Entrenar un modelo en AI Platform
gcloud ai-platform jobs submit training my_job --package-path trainer/ --module-name trainer.task --region us-central1 --python-version 3.7 --runtime-version 2.1

Cloud IAM

Cloud IAM (Identity and Access Management) es un servicio que permite gestionar el acceso a los recursos de GCP de manera segura.

Características principales:

  • Control de acceso: Define usuarios, grupos y roles con permisos específicos.
  • Políticas: Crea políticas para controlar el acceso a los recursos.
  • Integración con otros servicios de GCP: Se integra con todos los servicios de GCP.

Ejemplo básico:

# Crear un usuario en IAM
gcloud iam service-accounts create my-service-account --display-name "My Service Account"
