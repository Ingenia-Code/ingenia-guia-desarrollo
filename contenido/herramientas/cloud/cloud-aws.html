Amazon Web Services (AWS)

Amazon Web Services (AWS) es una plataforma de servicios en la nube que ofrece una amplia gama de servicios para computación, almacenamiento, bases de datos, machine learning, y más. AWS es ampliamente utilizado por empresas de todos los tamaños para construir aplicaciones escalables y seguras. A continuación, se presenta una guía detallada sobre 10 de los servicios más populares de AWS, con ejemplos prácticos y casos de uso.

Servicios populares de AWS

Amazon EC2 (Elastic Compute Cloud)

Amazon EC2 proporciona capacidad de computación escalable en la nube. Permite lanzar y gestionar servidores virtuales (instancias) en minutos.

Características principales:

  • Escalabilidad: Aumenta o reduce la capacidad según la demanda.
  • Flexibilidad: Soporta múltiples sistemas operativos y configuraciones.
  • Integración: Se integra con otros servicios de AWS como S3, RDS, y Lambda.

Ejemplo básico:

# Lanzar una instancia EC2
aws ec2 run-instances --image-id ami-0abcdef1234567890 --instance-type t2.micro --key-name MyKeyPair

Amazon S3 (Simple Storage Service)

Amazon S3 es un servicio de almacenamiento de objetos que ofrece escalabilidad, disponibilidad de datos y seguridad.

Características principales:

  • Almacenamiento ilimitado: Almacena cualquier cantidad de datos.
  • Durabilidad y disponibilidad: Diseñado para una durabilidad del 99.999999999%.
  • Control de acceso: Permite gestionar permisos y políticas de acceso.

Ejemplo básico:

# Crear un bucket S3
aws s3 mb s3://my-bucket

# Subir un archivo
aws s3 cp myfile.txt s3://my-bucket/

Amazon RDS (Relational Database Service)

Amazon RDS es un servicio de bases de datos relacionales que facilita la configuración, operación y escalabilidad de bases de datos en la nube.

Características principales:

  • Soporte para múltiples motores: MySQL, PostgreSQL, Oracle, SQL Server, y más.
  • Escalabilidad: Aumenta o reduce la capacidad de la base de datos según la demanda.
  • Copias de seguridad automáticas: Realiza copias de seguridad y restauraciones automáticas.

Ejemplo básico:

# Crear una instancia de RDS
aws rds create-db-instance --db-instance-identifier mydbinstance --db-instance-class db.t2.micro --engine mysql --master-username admin --master-user-password password

Amazon Lambda

Amazon Lambda es un servicio de computación sin servidor que ejecuta código en respuesta a eventos y administra automáticamente los recursos de computación.

Características principales:

  • Sin servidor: No es necesario gestionar servidores.
  • Escalabilidad automática: Escala automáticamente según la demanda.
  • Integración con otros servicios: Se integra con S3, DynamoDB, API Gateway, y más.

Ejemplo básico:

// Función Lambda en Node.js
exports.handler = async (event) => {
    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};

Amazon DynamoDB

Amazon DynamoDB es una base de datos NoSQL completamente gestionada que ofrece rendimiento rápido y escalable.

Características principales:

  • Rendimiento rápido: Latencia de milisegundos para cualquier escala.
  • Escalabilidad automática: Ajusta automáticamente la capacidad según la demanda.
  • Sin servidor: No es necesario gestionar la infraestructura.

Ejemplo básico:

# Crear una tabla en DynamoDB
aws dynamodb create-table --table-name MyTable --attribute-definitions AttributeName=Id,AttributeType=N --key-schema AttributeName=Id,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5

Amazon SNS (Simple Notification Service)

Amazon SNS es un servicio de mensajería y notificaciones que permite enviar mensajes a múltiples suscriptores.

Características principales:

  • Notificaciones push: Envía notificaciones a dispositivos móviles.
  • Integración con otros servicios: Se integra con Lambda, SQS, y más.
  • Escalabilidad: Maneja millones de mensajes por segundo.

Ejemplo básico:

# Crear un tema en SNS
aws sns create-topic --name MyTopic

# Suscribirse a un tema
aws sns subscribe --topic-arn arn:aws:sns:us-east-1:123456789012:MyTopic --protocol email --notification-endpoint user@example.com

Amazon SQS (Simple Queue Service)

Amazon SQS es un servicio de colas de mensajes completamente gestionado que permite desacoplar y escalar microservicios, sistemas distribuidos y aplicaciones sin servidor.

Características principales:

  • Desacoplamiento: Permite que los componentes de la aplicación funcionen de manera independiente.
  • Escalabilidad: Maneja cualquier volumen de mensajes.
  • Fiabilidad: Garantiza la entrega de mensajes al menos una vez.

Ejemplo básico:

# Crear una cola en SQS
aws sqs create-queue --queue-name MyQueue

# Enviar un mensaje
aws sqs send-message --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/MyQueue --message-body "Hello, SQS!"

Amazon CloudFront

Amazon CloudFront es un servicio de red de entrega de contenido (CDN) que acelera la distribución de contenido estático y dinámico a los usuarios finales.

Características principales:

  • Rendimiento rápido: Entrega contenido con baja latencia y alta velocidad de transferencia.
  • Seguridad: Ofrece integración con AWS Shield para protección DDoS.
  • Integración con S3: Facilita la distribución de contenido almacenado en S3.

Ejemplo básico:

# Crear una distribución de CloudFront
aws cloudfront create-distribution --origin-domain-name mybucket.s3.amazonaws.com

Amazon VPC (Virtual Private Cloud)

Amazon VPC permite crear una red virtual aislada en la nube de AWS, donde puedes lanzar recursos de AWS en una red definida por el usuario.

Características principales:

  • Aislamiento: Crea una red privada virtual en la nube.
  • Control de acceso: Define subredes, tablas de rutas y gateways.
  • Integración con otros servicios: Se integra con EC2, RDS, Lambda, y más.

Ejemplo básico:

# Crear una VPC
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# Crear una subred
aws ec2 create-subnet --vpc-id vpc-0abcdef1234567890 --cidr-block 10.0.1.0/24

Amazon IAM (Identity and Access Management)

Amazon IAM es un servicio que permite gestionar el acceso a los servicios y recursos de AWS de manera segura.

Características principales:

  • Control de acceso: Define usuarios, grupos y roles con permisos específicos.
  • Políticas: Crea políticas para controlar el acceso a los recursos.
  • Integración con otros servicios: Se integra con todos los servicios de AWS.

Ejemplo básico:

# Crear un usuario en IAM
aws iam create-user --user-name MyUser

# Asignar una política a un usuario
aws iam attach-user-policy --user-name MyUser --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess
